Scheduling Concepts / Glossary


To understand the concept of Scheduling it is a good idea to understand some of the terms that are used. Below is a table of the terms and an explanation of the term.

Multiprogramming The ability to run more processes/threads than the cpu has cores
Dispatcher Short term Scheduler. Please see the section on Short term Scheduler.
Job Scheduler Long term Scheduler. Please see the section on Long term Scheduler.
Quantum The slice of time a process has used before a context switch has occured.
Turnaround time The interval of time between process submission and completion. This is the sum of executing time, waiting time(Processor + I/O)
Response time Amount of time it takes from when a request was submitted until the first response is received.
Deadline The time a process must complete by.
Context Switch This is what occurs when the processor gives up the process.
I/O Request A process requests to do an Input/Output operation such as read a sector from a hard drive.
Blocked When a process makes an I/O operation request it enters a blocked state because it is waiting for some I/O to occur. It will be blocked until the I/O request is complete.
Ready A process wants to execute on the CPU. It is currently waiting in the ready queue and has not been scheduled to execute yet.
Running A CPU is executing the processes instructions
Terminated The OS has aborted a process
Process state transition A process will change its state from running to blocked to ready. Going from one to the other is called a state transition
CPU Burst The time that a process has on the CPU
IO Bound Process A process that performs lots of IO Operations
CPU bound Process A process that performs lots of computation and little I/O
