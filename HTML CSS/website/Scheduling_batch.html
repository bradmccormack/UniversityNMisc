Batch/ Non-Preemptive Scheduling


In non-preemptive scheduling a process will execute until it is finished completing its CPU burst.

(Er. Vivek Sharma, Er. Manish Varshney, Shantanu Sharma 2010, 125)


First Come First Served (FCFS)/ First In First Out (FIFO)


This is one of the simplest scheduling algorithms. Requests are serviced in the order that they are placed.

(A Dhotre 2009, 5-4)

FCFS does not guarantee good through put. It also does not provide the fastest service. There are no optimizations to reduce seek time. The major advantage of FCFS is that the algorithm is easy to implement.

(I.A Dhotre 2009, 5-4)

When the first job enters the system it is allowed to run as long as it wants to until it is complete. It is not pre-empted if it has run to long.

(Andrew S Tanenbaum 3e 2009, 150).

As jobs come in they are put onto the end of the queue. When the running process blocks the next process is started from the head of the queue.

(Andrew S Tanenbaum 3e 2009, 150)

It is also possible to do the reverse. You could add a new process to the head of the queue and get the next process from the tail, as long as the next process to be retrieved is the first process that was added.
